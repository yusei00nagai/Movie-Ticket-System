# Movie Ticket System

## 1. プロジェクト概要

### 1.1 目的
映画チケット購入の一連の業務フローを網羅したWebアプリケーション。以下の技術習得を目的とした学習用アプリ。

- **業務理解**: チケット予約における在庫管理やステータス遷移の理解
- **技術実践**: Spring Boot を用いたサーバーサイド開発、認証、DB設計の実践
- **高度な制御**: 非同期更新、排他制御（仮押さえ）、バッチ処理の習得
- **環境**: ローカル環境での動作を前提とする

### 1.2 システムの特徴
- **仮押さえ機能**: 座席選択時の競合を防ぐため、一定時間の占有を許容する
- **リアルタイム性**: 座席状態はフロントエンドから数秒おきにAPIを叩き、非同期で更新する
- **決済簡略化**: 学習用のため実決済機能は持たせず、ダミー処理とする

## 5. 画面遷移フロー

### メイン購入フロー（ゲスト・会員共通）
```
[映画一覧 / 上映スケジュール(全体) / 検索結果]
    ↓
[映画詳細]（上映スケジュール個別）
    ↓
【ログインチェック】 → （未ログイン）→ [ログイン / 新規登録]
    ↓ *1
[座席選択]（リアルタイム更新） → [予約エラー]（選択不可等）
    ↓
[購入内容確認]（仮押さえ: 1分間） → [予約エラー]（期限切れ等）
    ↓
[購入完了]（予約番号発行）
```

### 認証・会員管理フロー
```
[ログイン入力] ←→ [新規登録入力]
    ↓               ↓ 
[ログイン完了] →*1[座席選択]
    ↓
[TOPページ]

[マイページ（会員トップ）]
    ├→ [購入履歴一覧] → [予約詳細（チケット表示）]
    └→ [会員情報変更入力] → [会員情報変更完了]
*1：リダイレクト
```

### 管理者フロー
```
[管理者ログイン]
    ├→ [映画管理（一覧）] → [登録確認] → [完了]
    ├→ [劇場管理（一覧）] → [新規/編集入力] → [登録確認] → [完了]
    ├→ [スクリーン管理（一覧）] → [新規/編集入力] → [完了]
    ├→ [座席配置管理（一覧）] → [レイアウト編集] → [完了]
    ├→ [スケジュール管理（一覧）] → [新規/編集入力] → [完了]
    ├→ [予約状況モニタリング]
    └→ [会員管理一覧]
```

## 6. データ設計

### テーブル一覧
| テーブル | 説明 |
|----------|------|
| users | ユーザー認証と権限管理 |
| movies | 映画の基本情報（11件登録済み） |
| theaters | 劇場の基本情報（20館登録済み / TOHOシネマズ全国） |
| screens | 劇場に紐づくスクリーン情報（184件登録済み） |
| seats | スクリーン内の座席情報（複合ユニーク制約: screen_id, seat_number） |
| showtimes | 上映スケジュール（映画ID、スクリーンID、開始・終了時間） |
| reservations | 予約・仮押さえ管理（複合ユニーク制約: showtime_id, seat_id） |

### reservationsのステータス
| status | 意味 |
|--------|------|
| TEMPORARY | 仮押さえ中（expires_at まで有効） |
| CONFIRMED | 購入確定済み |

## 7. 業務ルール

### 仮押さえ
- 購入確認画面遷移時に `status='TEMPORARY'`, `expires_at=現在+1分` でINSERT
- バッチ処理（1分ごと）で期限切れのTEMPORARYレコードを削除

### 購入ルール
- 先着順（1つの上映回に対し1つの座席は1件のみ）
- 購入制限（1回の操作で最大5席まで）
- 決済はダミー（確定ボタン押下で予約確定）

### 非同期処理
- 座席選択画面にて数秒おきにポーリングし、座席状態をリアルタイムで可視化
